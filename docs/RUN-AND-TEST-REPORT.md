# Отчёт о запуске и тестировании (16.03.2025)

## Запуск проекта

### Что сделано

1. **backend/.env** — создан файл (его не было в репозитории) с переменными:
   - `DATABASE_URL=postgresql://minicrm:minicrm_secret@db:5432/minicrm`
   - `SECRET_KEY`, `ADMIN_USERNAME`, `ADMIN_PASSWORD` и др.

2. **Исправления при запуске:**
   - **migrations/env.py** — используется `DATABASE_URL` из окружения при наличии (для `docker compose exec backend alembic upgrade head`).
   - **backend/requirements.txt** — добавлено ограничение `bcrypt<5.0.0` из-за несовместимости passlib с bcrypt 5.x (ValueError при старте).
   - **auth.py** — добавлена защита от пароля длиннее 72 байт для bcrypt.
   - Миграции: БД уже содержала таблицы → выполнено `alembic stamp head`.

3. **Сервисы:**
   - **PostgreSQL** — контейнер `res-crm-db-1`, порт 5432.
   - **Backend** — контейнер `res-crm-backend-1`, порт 8000, Uvicorn с reload.
   - **Frontend** — `npm run dev` в папке `frontend`, http://localhost:3000/.

### Логи при запуске

- **Backend:** старт без ошибок. Есть неблокирующее предупреждение: `(trapped) error reading bcrypt version` (passlib и bcrypt 4.3) — на работу не влияет.
- **Frontend:** сборка `npm run build` и dev-сервер запускаются без ошибок.

## Запуск тестов backend (pytest)

В Docker-образ backend тестовые зависимости не установлены. Для запуска pytest локально:

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate   # Windows
pip install -r requirements-test.txt
set DATABASE_URL=sqlite:///:memory:
pytest
```

В CI тесты выполняются с `DATABASE_URL=sqlite:///:memory:` (см. .github/workflows/ci.yml).
