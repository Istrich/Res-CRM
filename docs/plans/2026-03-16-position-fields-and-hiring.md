# План: Поля позиции и вкладка Найм

**Дата:** 2026-03-16  
**Тип:** Новая функция

## Цель
- При добавлении позиции указывать: плановая дата выхода, оклад, проект, ставка в проекте.
- При создании позиции автоматически добавлять её в выбранный проект (с месяца выхода до конца года) и создавать записи оклада.
- Во вкладке «Найм» показывать эту информацию и статусы позиции; при найме конвертировать позицию в сотрудника (ФИО и т.д.), позиция пропадает из списка найма.

## Изменения

### 1. Модель и БД
- **employees**: добавить колонки (nullable, для позиций):
  - `planned_exit_date` (Date) — плановая дата выхода
  - `position_status` (String(50)) — статус: `awaiting_assignment` | `hiring` | `awaiting_start`
  - `planned_salary` (Numeric(15,2)) — плановый оклад (для отображения и создания SalaryRecord)
- Миграция: `0004_position_planned_fields.py` (revises 0003).

### 2. Backend API
- **EmployeeCreate**: при `is_position=True` опционально принимать `planned_exit_date`, `project_id`, `rate`, `planned_salary`. При указании — создавать Employee, один EmployeeProject (valid_from = 1-й день месяца planned_exit_date, valid_to = 31.12 этого года), SalaryRecord на каждый месяц с planned_exit_date по декабрь (salary = planned_salary).
- **EmployeeOut / EmployeeListItem**: добавить `planned_exit_date`, `position_status`, `planned_salary`.
- **EmployeeUpdate**: добавить `position_status` (для смены статуса позиции).
- **POST /employees/{id}/hire**: тело `{ first_name, last_name, middle_name, hire_date?, department?, specialization?, comment? }`. Только для `is_position=True`. Установить `is_position=False`, задать переданные поля; обнулить `planned_exit_date`, `position_status`, `planned_salary`. Позиция перестаёт отображаться в Найме.

### 3. Frontend
- **EmployeeForm**: при включённом «Позиция» показывать поля: Плановая дата выхода, Оклад, Проект (select из getProjects), Ставка в проекте. Валидация: для позиции при создании можно требовать эти поля или оставить опциональными (по ТЗ — указываем при добавлении).
- **HiringPage**: колонки — Статус, Должность, Подразделение, Специализация, Плановая дата выхода, Оклад, Проект, Ставка; фильтр по статусу; кнопка «Нанять» в строке (переход в карточку или вызов модалки).
- **EmployeeDetailPage**: для позиции показывать блок с плановой датой, окладом, проектом, ставкой и статусом; кнопка «Нанять» → модалка (ФИО, дата найма и т.д.) → вызов hire API → инвалидация и редирект/обновление (карточка станет сотрудником, из Найма пропадёт).

### 4. Статусы позиции (значения и подписи)
| Код | Подпись |
|-----|--------|
| awaiting_assignment | Ожидает взятия в работу |
| hiring | Найм |
| awaiting_start | Ожидаем выход |

## Риски и откат
- Миграция обратима (drop columns). Старые позиции без полей — отображать «—», статус по умолчанию `awaiting_assignment` или `hiring`.
- Не менять структуру ChromaDB, карточек, prompts.json, DI, network_mode.

## Проверка
- Создание позиции с датой/проектом/ставкой/окладом → в проекте назначение с месяца выхода по 31.12, зарплаты на те же месяцы.
- Вкладка Найм: видны все поля и статус; смена статуса; «Нанять» заполняет ФИО и дату, позиция становится сотрудником и исчезает из списка Найм.
