# План: критичные проблемы фронтенда

**Дата:** 2026-03-16  
**Слой:** Frontend (React, Vite)  
**Тип:** Рефакторинг без изменения поведения.

## Цели

1. Разбить `EmployeeDetailPage.jsx` (~700 строк) и вынести локальные компоненты `SalaryModal`, `AddAssignmentModal` в `components/*`, чтобы не терять их локальное состояние при рендерах страницы.
2. Убрать антипаттерн `JSON.stringify(initial)` в `EmployeeForm.jsx`, перейти на зависимость `[initial]` с использованием `key` в родителе.
3. Заменить нативный `confirm()` в `AssignmentManager.jsx` на общий компонент `Confirm` и унифицировать UX.
4. Вынести `parseImportTable` (и разбор дат) из `EmployeesPage.jsx` в `utils` для повторного использования и последующего покрытия тестами.

## Риски и откат

- Локальные компоненты модалок после выноса могут потерять доступ к состоянию/мутациям страницы 
  
  → минимизируем: все зависимости передаются через пропсы.
- Поведение `EmployeeForm` при смене `initial` может измениться (эффект будет срабатывать чаще) 
  
  → это допустимо, так как форма должна сбрасываться при смене сотрудника/режима.
- Перенос `parseImportTable` не должен менять формат ожидаемых данных; при ошибке импорта страница сразу покажет проблему.

Откат: revert коммита с фронтовыми изменениями; бэкенд не затрагивается.

## Шаги

### 1. EmployeeDetailPage: вынос модалок

- Создать `components/SalaryModal.jsx` и `components/AddAssignmentModal.jsx`, перенести туда соответствующие компоненты из низа `EmployeeDetailPage.jsx`.
- В `EmployeeDetailPage.jsx`:
  - Заменить локальные объявления `SalaryModal` и `AddAssignmentModal` на импорты.
  - Оставить только состояние/колбеки и использование этих компонентов.

### 2. EmployeeForm: зависимость useEffect

- В `EmployeeForm.jsx` заменить `useEffect` с зависимостью `JSON.stringify(initial)` на `useEffect(() => setForm({ ...EMPTY, ...initial }), [initial])`.
- В местах использования (как минимум в `EmployeeDetailPage.jsx`) при необходимости использовать `key={id}` для принудительного сброса формы при смене сотрудника.

### 3. AssignmentManager: Confirm и UX

- В `AssignmentManager.jsx` добавить импорт `Confirm` и состояние `deleteTarget`.
- Кнопку «Убрать» переписать на открытие `Confirm` с сообщением; само удаление выполнять в `onConfirm`.

### 4. parseImportTable 

- Создать `src/utils/parseImportTable.js` и вынести туда `parseImportDate` и `parseImportTable` из `EmployeesPage.jsx`.
- В `EmployeesPage.jsx` оставить `IMPORT_HEADERS` для UI, импортировать `parseImportTable` из `../utils/parseImportTable`.
- (Отдельная задача) Добавить Jest/Vitest тесты для `parseImportTable` при появлении тестового контура на фронте.

## Проверка (Tester)

- Открытие/редактирование сотрудника и позиции: модалки зарплаты, назначения и найма работают как раньше, состояние не теряется при переключении между сотрудниками.
- Форма сотрудника сбрасывается при открытии другого сотрудника или создании нового.
- В AssignmentManager при удалении использует стилизованный Confirm, нет нативных диалогов.
- Импорт сотрудников через текст/Excel работает как раньше (количество созданных/пропущенных записей, превью).
