# Перенос кнопки «Пересчитать» на Дашборд (2026-03-25)

## Тип задачи
Багфикс / UX-исправление: пользователь потерял доступ к кнопке пересчёта; логичнее держать её на странице дашбордов.

## План (шаги)
1. Убедиться, что кнопка `↻ Пересчитать` не дублируется в `BudgetsPage`.
2. Перенести кнопку на `DashboardPage` и подключить те же backend API (`POST /budgets/recalculate`).
3. После успешного пересчёта инвалидацией обновить данные всех dашбордов (query keys для `dashboard-*` и `last-calculated`).
4. Откат/rollback:
   - Удалить обработчики/вставку кнопки из `DashboardPage`.
   - Вернуть кнопку и мутацию обратно в `BudgetsPage`.
   - Вернуть пункт меню “Бюджеты” в сайдбар (если он окажется нужен).

## Риски
1. Несоответствие query keys: если забыть инвалидацию какого-то графика, часть дашбордов может обновиться не сразу.
2. Дублирование: если не удалить кнопку из `BudgetsPage`, получится 2 “Пересчитать”.
3. Нагрузка: пересчёт может быть тяжёлым; кнопка должна быть disabled во время выполнения.

## Как проверить
1. Открыть `/dashboard` и убедиться, что видна кнопка `↻ Пересчитать`.
2. Нажать кнопку и дождаться завершения.
3. Убедиться, что после пересчёта обновились значения на всех вкладках дашборда (`Обзор`, `Бюджетные проекты`, `Проекты`, `Подразделения`, `Специализации`, `Часовые ставки`).
4. Убедиться, что в `BudgetsPage` кнопки нет (только экспорт/таблицы).
