## План/отчет: пересобрать и перезапустить backend + frontend

**Дата:** 2026-03-18  
**Цель:** пересобрать бекенд (Docker) и фронтенд (Vite), затем перезапустить сервисы и подтвердить работоспособность healthcheck.

### Architect — план
1. Остановить текущий dev-сервер фронтенда (порт `3000`), чтобы избежать конфликтов/дублирования.
2. Пересобрать backend контейнер: `docker compose up -d --build backend`.
3. Дождаться готовности backend по `GET http://127.0.0.1:8000/health` (ожидается `200`).
4. Пересобрать фронтенд: `npm run build` в `frontend/`.
5. Поднять фронтенд dev-сервер: `npm run dev` и проверить `GET http://127.0.0.1:3000/` (ожидается `200`).

### Риски и rollback
1. В момент перезапуска возможна кратковременная недоступность `:8000`.
2. Если после пересборки backend упадет:
   - проверить логи контейнера `docker compose logs backend`;
   - выполнить повторный старт `docker compose up -d backend` (без необходимости отката к прежнему образу).
3. Если фронтенд не поднимается:
   - проверить консоль терминала `npm run dev`;
   - убедиться, что порт `3000` свободен.
