## Цель

Добавить отладочный/тестовый механизм, который проставляет всем текущим сотрудникам зарплаты и бонусы на весь год, чтобы в UI сразу было с чем работать.

## Область изменений

- Backend:
  - `employees_service.py` — функция генерации годовых записей `SalaryRecord` для всех сотрудников.
  - `employees.py` — debug-endpoint для запуска генерации.

## Детали реализации

- Новый сервис:
  - Функция принимает:
    - `db: Session`
    - `year: int`
  - Алгоритм:
    - Выбрать всех сотрудников, у которых `is_position = False`.
    - Для каждого сотрудника:
      - Для месяцев 1..12:
        - Если записи `SalaryRecord` на этот год/месяц нет — создать.
        - Схема сумм (чтобы были разные кейсы):
          - Базовый оклад зависит от индекса сотрудника (например, 80–200k).
          - Для части сотрудников `kpi_bonus` > 0 каждый месяц.
          - Для части сотрудников постоянный `fixed_bonus` > 0 каждый месяц.
          - Для части сотрудников разовые `one_time_bonus` в отдельных месяцах (например, квартальные премии).
    - Не трогаем уже существующие записи, чтобы не портить реальные данные.

- Новый endpoint:
  - `POST /employees/debug/seed-salaries`
  - Параметры:
    - `year: int` (query), по умолчанию — текущий год.
  - Access:
    - Только при `DEBUG_MODE=true` (как в `DELETE /employees/all`).
  - Возвращает:
    - Количество обработанных сотрудников и созданных записей.

## Риски и откат

- Риски:
  - Можно случайно вызвать на «живой» инстанс без DEBUG; нивелируем проверкой `DEBUG_MODE`.
  - Массовое создание записей — нагрузка на БД, но в dev/test масштабы небольшие.
- Откат:
  - Удалить endpoint и сервисную функцию.
  - Для уже созданных данных — использовать существующий `DELETE /employees/all` в dev либо SQL `DELETE FROM salary_records WHERE year = :year`.

## Шаги внедрения

1. Добавить сервисную функцию генерации зарплат в `employees_service.py`.
2. Добавить debug-endpoint в `employees.py`, использующий сервис.
3. Прогнать линтеры/тесты backend (минимум — `pytest backend/tests/test_api_employees.py::TestSalaryRecords`).
4. В dev окружении вызвать endpoint и визуально проверить в UI отображение зарплат, KPI, фиксированных и разовых премий.
