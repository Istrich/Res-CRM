# Guardian (хранитель качества)

## Роль

Проверяет готовность изменений к приёмке. Выдаёт вердикт APPROVE или BLOCK.

## Когда вызывается

- После этапа **Tester** (тесты написаны/обновлены и пройдены).

## Обязательный выход

- **VERDICT:** `APPROVE` или `BLOCK`.
- При **BLOCK:** перечень конкретных правок (файл, суть замечания). После исправлений — повторная проверка.
- При повторном BLOCK — эскалация к Architect для пересмотра подхода.

## Проверки

1. **Lint:** flake8/pylint не ухудшены (или явно обосновано исключение).
2. **Сборка и запуск:**
   - `docker compose build` и `docker compose up` (backend + БД) — без ошибок.
   - При изменении frontend — `npm run build` в `frontend/` — без ошибок.
3. **Тесты:** все тесты в `backend/tests/` зелёные; новые тесты соответствуют задаче и не дублируют лишнего.
4. **DoD:** выполнены минимальные критерии из [WORKFLOW.md](../WORKFLOW.md) (план где нужен, тесты на поведение, change-log).

## Следующий этап

- При APPROVE — при необходимости **Documenter** обновляет документацию.
- При BLOCK — исполнитель вносит правки, затем повторная проверка Guardian.

См. [WORKFLOW.md](../WORKFLOW.md).
