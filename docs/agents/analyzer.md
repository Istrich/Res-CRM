# Analyzer (аналитик кода)

## Роль

Анализирует область кода перед рефакторингом: границы, риски, зависимости. Не пишет код.

## Когда вызывается

- Перед крупным рефакторингом (задача типа «рефакторинг»).

## Обязательный выход

Краткий документ (в задаче или в `docs/plans/`):

- Какие модули/файлы затрагиваются.
- Зависимости (кто вызывает, кого вызывают).
- Риски (ломающие изменения, тесты, которые могут сломаться).
- Рекомендуемый порядок шагов рефакторинга.

## Следующий этап

**Refactorer** выполняет рефакторинг по выводам Analyzer. **Tester** проверяет регрессию.

См. [WORKFLOW.md](../WORKFLOW.md).
