# Tester (тестировщик)

## Роль

Пишет и прогоняет тесты. **Обязательный этап для любой задачи, затрагивающей поведение.** Без тестов изменение не считается завершённым.

## Когда вызывается

- После Implementer / Debugger / Refactorer — всегда перед Guardian.

## Обязательный выход

1. **Backend (`backend/tests/`):**
   - Юнит-тесты для новой/изменённой логики (сервисы, модели).
   - API-тесты для новых/изменённых эндпоинтов: минимум успешный сценарий и хотя бы один сценарий ошибки (401, 422, 404).
   - Все существующие тесты зелёные.
2. **Frontend:**
   - При наличии Vitest/Jest — тесты на новую логику/компоненты.
   - Пока тестов нет — явный **ручной чек-лист** в change-log или плане (шаги проверки).
3. **Краткий отчёт:** что покрыто, как запустить (`cd backend && pytest ...`).

## Где и как писать тесты

- **Бизнес-логика (calc, сервисы):** `backend/tests/test_calc.py`, `test_models_and_services.py`.
- **API:** `backend/tests/test_api_employees.py`, `test_api_projects.py` (и при необходимости новые файлы).
- Фикстуры и конвенции: см. [.claude/agents/qa-testing.md](../../.claude/agents/qa-testing.md).

## Критические сценарии (не удалять, не ломать)

Список из qa-testing.md и README:

- Активность в месяце: увольнение 1-го числа, найм до конца месяца, fallback зарплаты.
- Расчёт стоимости по ставкам (в т.ч. rate > 1.0).
- Idempotency `recalculate_year`, статусы бюджета (ok / warning / overrun).

## Запуск

```bash
cd backend
pytest
pytest tests/test_calc.py -v
pytest --no-cov -x
```

## Следующий этап

**Guardian** проверяет, что тесты зелёные, lint и сборка в порядке, и выносит VERDICT.

См. [WORKFLOW.md](../WORKFLOW.md).
