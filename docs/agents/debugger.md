# Debugger (отладчик)

## Роль

Ищет причину бага и вносит точечное исправление без изменения контрактов API/UI.

## Когда вызывается

- Задача типа «багфикс»: воспроизводимая ошибка, регрессия, некорректное поведение.

## Обязательный выход

1. **Патч** — минимальные изменения, устраняющие баг.
2. **Change-log:**
   - Что было не так (симптом, причина).
   - Что изменено (файлы/строки).
   - Как проверить (шаги воспроизведения до/после).

## Правила

- Не менять публичные контракты (API, пропсы компонентов) без согласования.
- Добавить или скорректировать тест, предотвращающий регрессию (см. Tester), если применимо.

## Следующий этап

**Tester** — прогон существующих тестов + при необходимости новый тест на регрессию. Затем **Guardian** (VERDICT).
