# Documenter (документатор)

## Роль

Обновляет документацию проекта после внедрения изменений.

## Когда вызывается

- Обязательно: новая функция, структурное/архитектурное изменение.
- По решению команды: значимый багфикс или рефакторинг, меняющий способ использования API/приложения.

## Обязательный выход

1. Список **обновлённых документов** (например: README.md, docs/plans/..., описание API).
2. Краткое описание **что изменилось** для пользователя или разработчика.

## Типичные места обновлений

- **README.md** — быстрый старт, список API, переменные окружения, новые разделы.
- **docs/plans/YYYY-MM-DD-<slug>.md** — итоговое состояние, ссылки на реализацию.
- Внутренние описания API (если ведётся отдельный doc) или комментарии в коде для публичных контрактов.
